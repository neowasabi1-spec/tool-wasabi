feat(mcp): async swipe jobs (up to 6h) + bump sync timeouts to Vercel max

The bot was hitting Vercel's 90-300s function timeout when swiping long
landing pages (e.g. Nooro advertorials). True 6h timeouts are impossible
in a single HTTP request on Vercel, so add an async pattern instead.

Sync path:
- /api/landing/swipe maxDuration: 120 -> 300 (Vercel Pro max)
- /api/mcp maxDuration: explicitly 300
- MCP swipe_landing_page* AbortSignal: 180s -> 290s

Async path (new):
- swipe_landing_page_async: enqueues job in openclaw_messages
  (section='swipe_job'), returns job_id immediately
- swipe_status: poll a job by id, optional blocking wait_seconds
- list_swipe_jobs: list recent async jobs

Worker changes (openclaw-worker.js):
- Detects section='swipe_job' messages and executes them by calling the
  tool API directly (HTTPS) with a 6-hour timeout. Runs on user's PC,
  so completely bypasses Vercel function limits.
- Supported actions: swipe_landing_page, clone_funnel, agentic_swipe,
  invoke_api (generic escape hatch).
- TOOL_BASE_URL env var (default: production URL).

Setup docs updated to describe the dual-mode worker.
